## เรียนรู้ Git & GitHub
ไม่จำเป็นต้องเป็นสายเดฟ(Developer) เท่านั้น ที่ใช้ Git หรือ GitHub ได้ แม้นักเรียนทั่วไป ก็ควรเรียนรู้ด้วย เหตุผล อาจจำแนกเป็นด้านๆ ได้ ดังนี้

1. ใช้สร้าง Portfolio สำหรับเก็บสะสมผลงาน
	GitHub เป็นพื้นที่โชว์ผลงานที่คุณเขียนจริงๆ บริษัทชั้นนำมักขอดูหน้าโปรไฟล์ GitHub มากกว่าดูเกรดเฉลี่ยซะอีก เพราะเขาสามารถเห็นความสม่ำเสมอของการทำงาน และคุณภาพของงานเราได้ทันที
	
2. แก้ปัญหา "ไฟล์หาย" หรือ "พังแล้วแก้กลับไม่ได้"
 Git คือ ระบบที่ทำหน้าที่เหมือนเครื่องย้อนกาลเวลา หรือ "Time Machine" ถ้าเราได้ทำงานอะไรบางอย่าง แล้วเกิดเบลอขึ้นมา ไม่ว่าจะงานใดๆ รายเขียนโค้ด เขียนรายงาน เมื่อแก้งานแล้ว มันเกิดพัง หรือไปต่อไม่ถูก  เราสามารถย้อนกลับไปยังเวอร์ชันที่ยังทำงานได้ปกติเพียงไม่กี่คลิก ไม่ต้องมานั่งเซฟไฟล์ชื่อ สมบูรณ์_1, สมบูรณ์_2 , สมบูรณ์สุด  อีกต่อไป 
 
3. ฝึกทำงานร่วมกับคนอื่น (Teamwork) 
 เวลาทำโปรเจกต์กลุ่ม Git จะช่วยให้คุณและเพื่อนช่วยกันแก้โค้ด หรือเนื้องานต่างๆ (ที่เป็นข้อความ) คนละส่วนในไฟล์เดียวกันได้ โดยที่เนื้อหาของงาน หรือ โค้ดไม่ทับซ้อนกันจนพัง นี่เป็นทักษะที่บริษัททั่วโลก 95% ใช้งานจริงในการทำงาน 
 
4. สิทธิพิเศษสำหรับนักเรียน (GitHub Education) 
 เมื่อสมัครผ่าน GitHub Education ด้วยอีเมลสถานศึกษา นักเรียนจะได้รับ Student Developer Pack ซึ่งรวมเครื่องมือพรีเมียม ที่ปกติคนทั่วๆ ไปต้องเสียตังค์ แต่นักเรียนใช้ฟรี เช่น เครดิต Cloud, เครื่องมือออกแบบ รวมถึงคอร์สเรียนต่างๆ ด้วย

## การใช้ GitHub-Cli
*คำสั่ง,การใช้งาน
gh auth login,สำคัญที่สุด: ใช้สำหรับ Login และตั้งค่าสิทธิ์การเข้าถึง
gh auth status,ตรวจสอบว่าตอนนี้คุณ Login ค้างไว้ด้วย User ไหน
gh repo create,สร้าง Repository ใหม่บน GitHub จาก Terminal ได้เลย
gh repo clone <name>,Clone repo ลงเครื่อง (จัดการเรื่องสิทธิ์ให้อัตโนมัติ)
gh pr create,สร้าง Pull Request โดยไม่ต้องเปิด Browser
gh browse,เปิดหน้าเว็บของ Repo ปัจจุบันบน Browser ทันที
